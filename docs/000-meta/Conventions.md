# Atlas Engineering Conventions

## Session Rules

Every working session should produce at least one tangible artifact.

Before ending a session:

- [ ] At least one Git commit has been created.
- [ ] Architectural decisions have been recorded as ADRs.
- [ ] New technical terms have been added to the Glossary.
- [ ] Open questions have been added to Questions.md.
- [ ] Atlas-Memory.md has been updated (if context changed).
- [ ] STATUS.md has been updated (if progress changed).

---

## Documentation Rules

- One document = one concept.
- Every document has a unique ID.
- Use Markdown.
- Prefer Mermaid for diagrams.
- Include Atlas Perspective where applicable.

---

## Engineering Rules

- Benchmark before optimizing.
- Build before abstracting.
- Keep designs simple until complexity is justified.
- Record important trade-offs in ADRs.
